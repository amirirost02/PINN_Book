فصل ۱۱: چالش اول — اعمال دقیق شرایط مرزی (از روش جریمه تا قید سخت)

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. محدودیت‌های روش جریمه (Penalty Method) را در اعمال شرایط مرزی توضیح دهید.
  2. روش جریمهٔ تطبیقی (Adaptive Penalty) را برای بهبود همگرایی پیاده‌سازی کنید.
  3. یک شرط مرزی را با استفاده از تابع فاصلهٔ علامت‌دار (SDF) به‌صورت سخت (Hard Constraint) اعمال کنید.
  4. با استفاده از فرمول‌بندی جمعی، هم شرایط مرزی و هم شرط اولیه را به‌صورت سخت اعمال کنید.
  5. دلیل استفاده از توان دوم زمان ($t^2$) در قید سخت شرایط اولیهٔ سرعت را به‌صورت ریاضی و شهودی تحلیل کنید.
  6. روش مناسب اعمال شرایط مرزی را بر اساس نوع مسئله و هندسهٔ دامنه انتخاب کنید.

روش جریمه: رویکردی ساده اما حساس

در تمام فصول گذشته، برای اعمال شرایط مرزی و اولیه از روش جریمه (Penalty Method) استفاده کردیم:

$$ \mathcal{L}_{total} = \mathcal{L}_{physics} + \lambda_{BC} \cdot \mathcal{L}_{BC} + \lambda_{IC} \cdot \mathcal{L}_{IC} $$

روش جریمه سه محدودیت اساسی دارد:

محدودیت حساسیت به وزن‌ها

اگر وزن $\lambda_{BC}$ بسیار کوچک باشد، شبکه شرط مرزی را نادیده می‌گیرد. اگر بسیار بزرگ باشد، شبکه تمام توان خود را صرف کاهش $\mathcal{L}_{BC}$ می‌کند.

محدودیت نمونه‌برداری از مرزها

روش جریمه فقط در نقاط مرزی که در تابع هزینه گنجانده شده‌اند، شرط را اعمال می‌کند.

محدودیت عدم تضمین فیزیکی

روش جریمه یک روش «نرم» است. شبکه مجاز است شرط مرزی را تا حدودی نقض کند.

روش جریمه: یک مثال نقض‌شده

برای درک محدودیت‌ها، مثال معادلهٔ لاپلاس یک‌بعدی با جواب تحلیلی $u(x) = x$ را بررسی کنید:

ضریب وزنی ($\lambda_{BC}$) خطای شرط مرزی خطای معادله نتیجه
۰.۰۱ زیاد کم جواب نادرست
۱.۰ متوسط متوسط جواب قابل‌قبول
۱۰۰.۰ بسیار کم زیاد جواب صاف و بی‌مفهوم
شکل ۱۱-۱: مقایسهٔ سه روش اعمال شرایط مرزی — جواب تحلیلی (خط آبی ممتد)، جریمه (خط قرمز چین)، و قید سخت (خط سبز نقطه‌چین)

روش جریمهٔ تطبیقی (Adaptive Penalty)

$$ \lambda_{BC}^{(epoch)} = \lambda_{BC}^{initial} \cdot \left( 1 + \frac{epoch}{N_{\max}} \right) $$
lambda_bc_initial = 1.0 lambda_bc_max = 100.0 N_epochs = 8000 for epoch in range(N_epochs): lambda_bc = lambda_bc_initial + (lambda_bc_max - lambda_bc_initial) * (epoch / N_epochs) loss = loss_physics + lambda_bc * loss_bc
شکل ۱۱-۲: نمایش تابع فاصلهٔ علامت‌دار (SDF) برای دامنهٔ یک‌بعدی $x \in [0,1]$ — تابع در مرزها صفر و در داخل مثبت است

راه‌حل پیشرفته: اعمال سخت شرایط مرزی با SDF

ایدهٔ اصلی: به‌جای جریمه کردن خطا در مرزها، شرط مرزی را در ساختار شبکه گنجانده می‌کنیم.

روش SDF (Signed Distance Function):

$$ u_{NN}(x,t) = SDF(x,t) \cdot \mathcal{N}(x,t) + g(x,t) $$

SDF برای دامنه‌های ساده

دامنهٔ یک‌بعدی $x \in [0, L]$: $SDF(x) = x(L - x)$

دامنهٔ مستطیلی: $SDF(x,y) = x(L_x - x) \cdot y(L_y - y)$

شرایط مرزی غیرصفر: $g(x,t) = a + (b - a)\frac{x}{L}$

ترکیب قیدهای سخت برای شرایط مرزی و اولیه (فرمول‌بندی جمعی)

$$ u_{NN}(x,t) = g(x,t) + SDF(x) \cdot h(t) \cdot \mathcal{N}(x,t) $$

مثال: انتقال حرارت با $T(0,t) = 0$، $T(1,t) = 0$، و $T(x,0) = \sin(\pi x)$:

  • $g(x,t) = \sin(\pi x)$ (شرط اولیه را برآورده می‌کند)
  • $SDF(x) = x(1 - x)$ (در مرزهای مکانی صفر است)
  • $h(t) = t$ (در $t=0$ صفر است)

خروجی نهایی:

$$ T_{NN}(x,t) = \sin(\pi x) + t \cdot x(1 - x) \cdot \mathcal{N}(x,t) $$

فلسفهٔ ریاضی و شهودی $t^2$ در قید سخت شرایط اولیهٔ سرعت

اگر مسئله شامل شرط اولیهٔ سرعت باشد ($\dot{x}(0) = v_0$):

$$ x_{NN}(t) = x_0 + v_0 \cdot t + t^2 \cdot \mathcal{N}(t) $$
  • در $t = 0$: $x_{NN}(0) = x_0$ ✓
  • مشتق: $\dot{x}_{NN}(t) = v_0 + 2t \cdot \mathcal{N}(t) + t^2 \cdot \dot{\mathcal{N}}(t)$
  • در $t = 0$: $\dot{x}_{NN}(0) = v_0$ ✓

چرا $t$ کافی نیست؟ با $x_{NN}(t) = x_0 + v_0 \cdot t + t \cdot \mathcal{N}(t)$، شرط سرعت به $v_0 + \mathcal{N}(0)$ تبدیل می‌شود که تنها با $\mathcal{N}(0) = 0$ برقرار است.

شکل ۱۱-۳: مقایسهٔ خطای L2 در سه روش اعمال شرایط مرزی — قید سخت عملکرد بهتری دارد

کد کامل PINN با قید سخت (SDF + فرمول‌بندی جمعی)

# ════════════════════════════════════════════════════════════ # بلوک ۱: تعریف شبکه با قید سخت کامل # ════════════════════════════════════════════════════════════ import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt torch.set_default_dtype(torch.float64) class PINN_HardConstraint(nn.Module): def __init__(self, layers=[2, 50, 50, 50, 50, 1]): super(PINN_HardConstraint, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'tanh_{i}', nn.Tanh()) def forward(self, x, t): g = torch.sin(np.pi * x) sdf = x * (1 - x) h = t u = self.net(torch.cat([x, t], dim=1)) return g + sdf * h * u # ════════════════════════════════════════════════════════════ # بلوک ۲: تولید نقاط کالوکیشن (فقط نقاط داخلی) # ════════════════════════════════════════════════════════════ L, T = 1.0, 1.0 alpha = 0.01 N_col = 5000 x_col = torch.rand(N_col, 1, requires_grad=True) * L t_col = torch.rand(N_col, 1, requires_grad=True) * T # ════════════════════════════════════════════════════════════ # بلوک ۳: تابع هزینه (فقط L_physics) # ════════════════════════════════════════════════════════════ model = PINN_HardConstraint() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) def compute_loss(): T_pred = model(x_col, t_col) T_t = torch.autograd.grad(T_pred, t_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_x = torch.autograd.grad(T_pred, x_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_xx = torch.autograd.grad(T_x, x_col, grad_outputs=torch.ones_like(T_x), create_graph=True, retain_graph=True)[0] residual = T_t - alpha * T_xx loss_physics = torch.mean(residual**2) return loss_physics # ════════════════════════════════════════════════════════════ # بلوک ۴: حلقه‌ی آموزش # ════════════════════════════════════════════════════════════ loss_history = [] n_epochs_adam = 8000 for epoch in range(n_epochs_adam): optimizer.zero_grad() loss = compute_loss() loss.backward() optimizer.step() loss_history.append(loss.item()) if (epoch+1) % 2000 == 0: print(f'Epoch {epoch+1}, Loss: {loss.item():.2e}') optimizer_lbfgs = torch.optim.LBFGS(model.parameters(), lr=1.0, max_iter=500) def closure(): optimizer_lbfgs.zero_grad() loss = compute_loss() loss.backward() return loss optimizer_lbfgs.step(closure) # ════════════════════════════════════════════════════════════ # بلوک ۵: ارزیابی # ════════════════════════════════════════════════════════════ x_test = torch.linspace(0, L, 100).reshape(-1, 1) t_test = torch.linspace(0, T, 100).reshape(-1, 1) X, T_grid = torch.meshgrid(x_test.squeeze(), t_test.squeeze(), indexing='ij') x_flat = X.reshape(-1, 1) t_flat = T_grid.reshape(-1, 1) with torch.no_grad(): T_pred_flat = model(x_flat, t_flat).numpy() T_pred = T_pred_flat.reshape(X.shape) T_exact = np.sin(np.pi * X.numpy()) * np.exp(-alpha * np.pi**2 * T_grid.numpy()) error_l2 = np.linalg.norm(T_pred - T_exact) / np.linalg.norm(T_exact) print(f'L2 Error: {error_l2:.2e}') fig, axes = plt.subplots(1, 2, figsize=(12, 5)) im1 = axes[0].contourf(T_grid, X, T_exact, levels=50, cmap='hot') axes[0].set_title('Analytical') plt.colorbar(im1, ax=axes[0]) im2 = axes[1].contourf(T_grid, X, T_pred, levels=50, cmap='hot') axes[1].set_title('PINN (Hard Constraint)') plt.colorbar(im2, ax=axes[1]) plt.tight_layout() plt.show()

جمع‌بندی و مسیر پیش‌رو

در این فصل، با سه رویکرد برای اعمال شرایط مرزی آشنا شدیم. نکات کلیدی:

  1. روش جریمه ثابت ساده است اما به تنظیم وزن حساس است.
  2. روش جریمه تطبیقی با افزایش تدریجی وزن، نیاز به تنظیم دستی را کاهش می‌دهد.
  3. روش SDF (قید سخت) با گنجاندن شرط مرزی در ساختار شبکه، آن را تضمین می‌کند.
  4. استفاده از $t^2$ به‌جای $t$ برای اعمال سخت شرط اولیهٔ سرعت ضروری است.

پل به فصل دوازدهم: در فصل بعد، با چالش انتخاب نقاط کالوکیشن و معماری شبکه (RAR و SIREN) آشنا خواهید شد.
