فصل ۱۴: چشم‌انداز پیشرفته — DeepONet، FNO، و B-PINN

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. مفهوم «یادگیری عملگرها» (Operator Learning) را در مقابل «یادگیری توابع» توضیح دهید.
  2. معماری DeepONet را تشریح کرده و یک پیاده‌سازی سادهٔ آن را در PyTorch اجرا کنید.
  3. تفاوت FNO (Fourier Neural Operator) با PINN و DeepONet را در حل معادلات دیفرانسیل بیان کنید.
  4. مفهوم B-PINN (Bayesian PINN) و روش MC-Dropout را برای تخمین عدم‌قطعیت در مسائل مهندسی پیاده‌سازی کنید.
  5. کاربردهای بالقوهٔ PINN در کنترل بهینه و طراحی بهینهٔ توپولوژی را شناسایی کنید.
  6. مسیر پژوهشی و منابع پیشرو را برای مطالعهٔ بیشتر در حوزهٔ PINN ترسیم نمایید.

چرا به «فراتر از PINN» نیاز داریم؟

PINN دو محدودیت اساسی دارد:

  1. حل تکی برای هر مسئله: هر بار که شرایط تغییر می‌کند، باید از ابتدا آموزش داد.
  2. پیش‌بینی قطعی: هیچ تخمینی از عدم‌قطعیت ارائه نمی‌دهد.

این فصل سه حوزهٔ پیشرفته را معرفی می‌کند:

حوزه سوال کلیدی کاربرد اصلی
یادگیری عملگرها چگونه یک بار شبکه را آموزش دهیم برای هر تابع ورودی؟ طراحی بهینه، تحلیل حساسیت
PINN بیزی چگونه عدم‌قطعیت پیش‌بینی‌ها را تخمین بزنیم؟ تصمیم‌گیری در شرایط عدم‌قطعیت
کنترل و طراحی بهینه چگونه از PINN برای کنترل بهینه استفاده کنیم؟ کنترل بهینه، بهینه‌سازی توپولوژی

یادگیری عملگرها — از حل یک مسئله تا حل خانواده‌ای از مسائل

در تمام فصول گذشته، شبکه‌های عصبی را برای یادگیری توابع به کار می‌گرفتیم. در یادگیری عملگرها، هدف یافتن نگاشتی از یک تابع به یک تابع دیگر است.

مثال: عملگر مشتق‌گیری $\frac{d}{dx}$ یک تابع را به تابع دیگر تبدیل می‌کند.

DeepONet: شبکه‌های عملگری عمیق

معماری DeepONet

شاخهٔ تابعی (Branch Network): مقادیر گسسته‌شدهٔ تابع ورودی را به بردار $b \in \mathbb{R}^p$ تبدیل می‌کند.

شاخهٔ مکانی (Trunk Network): مختصات نقطهٔ خروجی را به بردار $t \in \mathbb{R}^p$ تبدیل می‌کند.

ترکیب نهایی:

$$ u(x) = \sum_{i=1}^{p} b_i \cdot t_i = \langle b, t \rangle $$

کد DeepONet

class DeepONet(nn.Module): def __init__(self, m=20, p=50, hidden_size=64): super(DeepONet, self).__init__() self.branch_net = nn.Sequential( nn.Linear(m, hidden_size), nn.Tanh(), nn.Linear(hidden_size, hidden_size), nn.Tanh(), nn.Linear(hidden_size, p) ) self.trunk_net = nn.Sequential( nn.Linear(1, hidden_size), nn.Tanh(), nn.Linear(hidden_size, hidden_size), nn.Tanh(), nn.Linear(hidden_size, p) ) def forward(self, branch_input, trunk_input): b = self.branch_net(branch_input) t = self.trunk_net(trunk_input) output = torch.sum(b * t, dim=1, keepdim=True) return output

FNO (Fourier Neural Operator)

ایدهٔ اصلی: در هر لایه، داده‌ها با تبدیل فوریه به حوزهٔ فرکانس منتقل می‌شوند و یک فیلتر قابل‌آموزش روی مؤلفه‌های فرکانسی اعمال می‌شود.

معیار DeepONet FNO
رویکرد اصلی دو شاخه (تابعی + مکانی) تبدیل فوریه در لایه‌ها
پیچیدگی پیاده‌سازی ساده پیچیده (نیاز به FFT)
دقت در فرکانس‌های بالا خوب عالی
سرعت آموزش متوسط بالا (برای نقاط زیاد)
مناسب برای مسائل با تعداد نقاط کم مسائل با مش‌بندی ریز

B-PINN (Bayesian PINN): تخمین عدم‌قطعیت

چرا عدم‌قطعیت مهم است؟

PINN معمولی یک پیش‌بینی قطعی ارائه می‌دهد. B-PINN با ارائهٔ توزیع احتمالی، عدم‌قطعیت را مشخص می‌کند.

روش MC-Dropout

  1. لایه‌های Dropout را به معماری PINN اضافه می‌کنیم.
  2. در زمان ارزیابی، model.train() را فعال می‌کنیم.
  3. شبکه را $N$ بار (معمولاً $N = 100$) اجرا می‌کنیم.
  4. میانگین و انحراف معیار خروجی‌ها را محاسبه می‌کنیم.
شکل ۱۴-۲: تخمین عدم‌قطعیت با B-PINN — نوار اطمینان ۹۵\% (ناحیه خاکستری) در کنار پیش‌بینی میانگین (خط آبی) و داده‌های واقعی (نقاط قرمز)
class PINN_with_Dropout(nn.Module): def __init__(self, layers=[1, 50, 50, 50, 1], dropout_rate=0.1): super(PINN_with_Dropout, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'dropout_{i}', nn.Dropout(dropout_rate)) self.net.add_module(f'tanh_{i}', nn.Tanh()) def forward(self, x): return self.net(x) def mc_dropout_prediction(model, x_input, n_samples=100): model.train() predictions = [] with torch.no_grad(): for _ in range(n_samples): pred = model(x_input) predictions.append(pred) predictions = torch.stack(predictions, dim=0) mean = predictions.mean(dim=0) std = predictions.std(dim=0) return mean, std

PINN در کنترل بهینه و طراحی بهینهٔ توپولوژی

کنترل بهینه با PINN

دو روش اصلی:

  1. حل معادلهٔ HJB: PINN معادلهٔ هامیلتون-ژاکوبی-بلمن را حل کرده و سیگنال کنترل بهینه را استخراج می‌کند.
  2. PINN به‌عنوان جایگزین مدل: در کنترل پیش‌بینی مدل (MPC)، زمان محاسبات را کاهش می‌دهد.

طراحی بهینهٔ توپولوژی

PINN می‌تواند:

  • به‌عنوان حل‌کنندهٔ سریع تحلیل تنش در هر تکرار استفاده شود.
  • با تعریف پارامترهای طراحی به‌عنوان ورودی، بهینه‌سازی را در فضای پارامتری انجام دهد.
