فصل ۴: اولین PINN در PyTorch — حل ODE مرتبه اول

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. یک PINN کامل را در PyTorch برای حل یک معادلهٔ دیفرانسیل معمولی (ODE) مرتبهٔ اول پیاده‌سازی کنید.
  2. ساختار ۵ بلوکی کد PINN را تشریح کرده و نقش هر بلوک را در فرآیند آموزش تبیین نمایید.
  3. عملکرد پارامترهای grad_outputs، create_graph و retain_graph را در تابع torch.autograd.grad تحلیل کنید.
  4. تأثیر تعداد نقاط کالوکیشن و عمق شبکه را بر دقت و سرعت همگرایی بررسی نمایید.
  5. با تغییر پارامترهای شبکه، تحلیل حساسیتی ساده بر روی عملکرد PINN انجام دهید.

معرفی مسئله: یک سیستم ساده برای شروع

به‌عنوان اولین مسئلهٔ عملی، ساده‌ترین معادلهٔ دیفرانسیل معمولی را انتخاب می‌کنیم که جواب تحلیلی مشخصی دارد:

$$ \frac{du}{dt} = -u, \quad u(0) = 1, \quad t \in [0, 1] $$

این معادله، یک مدل سادهٔ واپاشی نمایی را توصیف می‌کند. جواب تحلیلی آن به‌صورت زیر است:

$$ u(t) = e^{-t} $$

هدف ما این است که با استفاده از یک PINN، تابع $u(t)$ را بدون هیچ دادهٔ داخلی (فقط با استفاده از شرط اولیه و معادلهٔ دیفرانسیل) در بازهٔ $[0, 1]$ بازسازی کنیم.

کد کامل PINN (نسخهٔ جامع)

# ════════════════════════════════════════════════════════════ # بلوک ۱: تعریف شبکه‌ی عصبی # ════════════════════════════════════════════════════════════ import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt class PINN(nn.Module): """شبکه‌ی عصبی برای حل ODE مرتبه‌ی اول""" def __init__(self, layers=[1, 50, 50, 50, 50, 1]): super(PINN, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'tanh_{i}', nn.Tanh()) def forward(self, t): return self.net(t) # ════════════════════════════════════════════════════════════ # بلوک ۲: تولید نقاط کالوکیشن و نقاط داده # ════════════════════════════════════════════════════════════ t0, T = 0.0, 1.0 u0 = 1.0 N_col = 1000 t_col = torch.rand(N_col, 1, requires_grad=True) * T t_ic = torch.tensor([[t0]], requires_grad=True) u_ic_true = torch.tensor([[u0]]) # ════════════════════════════════════════════════════════════ # بلوک ۳: تعریف تابع هزینه # ════════════════════════════════════════════════════════════ model = PINN() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) def compute_loss(t_col, t_ic): u_col = model(t_col) du_dt = torch.autograd.grad( u_col, t_col, grad_outputs=torch.ones_like(u_col), create_graph=True, retain_graph=True )[0] physics_residual = du_dt + u_col loss_physics = torch.mean(physics_residual**2) u_ic_pred = model(t_ic) loss_data = torch.mean((u_ic_pred - u_ic_true)**2) loss = loss_physics + loss_data return loss, loss_physics, loss_data # ════════════════════════════════════════════════════════════ # بلوک ۴: حلقه‌ی آموزش # ════════════════════════════════════════════════════════════ loss_history = [] n_epochs_adam = 8000 for epoch in range(n_epochs_adam): optimizer.zero_grad() loss, _, _ = compute_loss(t_col, t_ic) loss.backward() optimizer.step() loss_history.append(loss.item()) if (epoch + 1) % 2000 == 0: print(f'Epoch {epoch+1}/{n_epochs_adam}, Loss: {loss.item():.2e}') optimizer_lbfgs = torch.optim.LBFGS(model.parameters(), lr=1.0, max_iter=500) def closure(): optimizer_lbfgs.zero_grad() loss, _, _ = compute_loss(t_col, t_ic) loss.backward() return loss optimizer_lbfgs.step(closure) # ════════════════════════════════════════════════════════════ # بلوک ۵: ارزیابی و رسم نتایج # ════════════════════════════════════════════════════════════ t_test = torch.linspace(0, T, 500).reshape(-1, 1) with torch.no_grad(): u_pred = model(t_test).numpy() u_exact = np.exp(-t_test.numpy()) error_l2 = np.linalg.norm(u_pred - u_exact) / np.linalg.norm(u_exact) print(f'L2 Error: {error_l2:.2e}') fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 5)) ax1.plot(t_test, u_exact, 'b-', label='Analytical', linewidth=2) ax1.plot(t_test, u_pred, 'r--', label='PINN Prediction', linewidth=2) ax1.set_xlabel('Time $t$', fontsize=12) ax1.set_ylabel('$u(t)$', fontsize=12) ax1.set_title('Comparison: PINN vs Analytical Solution', fontsize=14) ax1.legend() ax1.grid(True, alpha=0.3) ax2.semilogy(loss_history) ax2.set_xlabel('Epoch', fontsize=12) ax2.set_ylabel('Loss', fontsize=12) ax2.set_title('Training Loss History', fontsize=14) ax2.grid(True, alpha=0.3) plt.tight_layout() plt.show()

رمزگشایی از پارامترهای Autograd در PyTorch

در کد بالا، از تابع torch.autograd.grad با سه پارامتر کلیدی استفاده شده است:

  • grad_outputs=torch.ones_like(u_col): تابع autograd.grad مشتق یک کمیت اسکالر را محاسبه می‌کند. اما خروجی شبکه یک بردار است. با استفاده از بردار یک‌ها، به PyTorch می‌گوییم که مجموع تمام مؤلفه‌ها را به‌عنوان اسکالر در نظر بگیرد.
  • create_graph=True: برای محاسبهٔ مشتق دوم، به گراف محاسباتی مشتق اول نیاز داریم. این پارامتر، گراف را برای استفادهٔ مجدد حفظ می‌کند.
  • retain_graph=True: پس از هر بار فراخوانی autograd.grad، PyTorch گراف موقتی را پاک می‌کند. این پارامتر از آزادسازی زودهنگام جلوگیری می‌کند.

جمع‌بندی و مسیر پیش‌رو

در این فصل، اولین PINN عملی خود را پیاده‌سازی کردیم. نکات کلیدی:

  1. الگوی ۵ بلوکی ثابت: کد PINN همیشه از ۵ بلوک تشکیل شده است و برای حل مسائل جدید، فقط بلوک سوم (تابع هزینه) تغییر می‌کند.
  2. مقایسه با NN معمولی: PINN با تنها ۱ نقطه داده (شرط اولیه)، به خطای $10^{-5}$ می‌رسد.
  3. پارامترهای Autograd: درک صحیح grad_outputs، create_graph و retain_graph برای جلوگیری از خطاهای گراف محاسباتی ضروری است.

تمرین‌های عملی

  1. تمرین ۱ (تأثیر تعداد نقاط کالوکیشن): کد را با تعداد نقاط کالوکیشن $N_{col} = 50$ و $N_{col} = 5000$ اجرا کنید و خطای L2 را ثبت کنید.
  2. تمرین ۲ (تغییر معماری شبکه): معماری شبکه را از ۴ لایهٔ پنهان (۵۰ نورون) به ۲ لایهٔ پنهان (۲۰ نورون) تغییر دهید و خطای L2 را با حالت اصلی مقایسه کنید.
  3. تمرین ۳ (پیاده‌سازی قید سخت): خروجی شبکه را از $u = net(t)$ به $u = 1 + t * net(t)$ تغییر دهید (به‌طوری که شرط اولیه تضمین شود). تابع هزینه را با حذف loss_data اصلاح کنید.
  4. تمرین ۴ (کاوش بیشتر): معادلهٔ دیفرانسیل $\frac{du}{dt} = -2u$ با شرط اولیه $u(0) = 2$ را در نظر بگیرید. کد را برای این معادله اصلاح کنید.
