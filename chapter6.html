فصل ۶: مسئلهٔ معکوس — شناسایی ضریب مجهول

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. یک پارامتر فیزیکی مجهول را به‌عنوان nn.Parameter در PyTorch تعریف کرده و آن را به‌طور هم‌زمان با شبکه آموزش دهید.
  2. یک مسئلهٔ معکوس را با استفاده از PINN برای شناسایی ضریب مجهول معادلهٔ دیفرانسیل حل کنید.
  3. مفهوم «عدم یکتایی» (Non-Uniqueness) را در مسائل معکوس توضیح داده و نقش داده‌ها را در رفع آن تحلیل نمایید.
  4. تأثیر تعداد داده‌های اندازه‌گیری، وجود نویز، و نوع داده‌ها (گذرا در برابر ماندگار) را بر دقت تخمین پارامتر تحلیل نمایید.
  5. تفاوت بین «مسئلهٔ مستقیم» و «مسئلهٔ معکوس» را در زمینهٔ PINN تشریح کنید.

از «حل معادله» تا «شناسایی پارامتر»: تغییر پارادایم

در دو فصل گذشته، مسائل مستقیم (Forward) را بررسی کردیم. در این نوع مسائل، معادلهٔ دیفرانسیل، ضرایب فیزیکی، و شرایط مرزی/اولیه کاملاً شناخته‌شده هستند و هدف، یافتن میدان مجهول در سراسر دامنه است.

اما در دنیای واقعی مهندسی — به‌ویژه در پایش سلامت سازه‌های ریلی — وضعیت اغلب برعکس است. داده‌های اندازه‌گیری (مثلاً شتاب چند نقطه از یک بوژی یا دمای چند سنسور روی یاتاقان‌ها) را در اختیار داریم، اما ضرایب فیزیکی ماده (مانند سختی فنرها، ضریب میرایی، یا ضریب نفوذ حرارتی) را نمی‌دانیم. این دسته از مسائل را معکوس (Inverse) می‌نامند.

شکل ۶-۱: مقایسهٔ شماتیک مسئلهٔ مستقیم (چپ) و مسئلهٔ معکوس (راست) در PINN

یک مثال ملموس در حوزهٔ راه‌آهن: فرض کنید چند شتاب‌سنج روی بوژی یک قطار نصب شده‌اند و داده‌های ارتعاشی هنگام عبور از مسیر ثبت شده‌اند. می‌خواهیم سختی فنرهای ثانویه و ضریب میرایی دمپرها را تخمین بزنیم.

چالش بنیادین مسائل معکوس: عدم یکتایی (Non-Uniqueness)
در یک مسئلهٔ معکوس، ممکن است چندین مجموعه پارامتر مختلف وجود داشته باشد که همگی معادلهٔ دیفرانسیل را ارضا کرده و با داده‌های محدود اندازه‌گیری تطابق داشته باشند. PINN با ترکیب تلفات فیزیک (که معادله را در کل دامنه اعمال می‌کند) و تلفات داده (که شبکه را به نقاط اندازه‌گیری مقید می‌کند)، این ابهام را برطرف می‌کند.

معرفی مسئله: شناسایی ضریب نفوذ حرارتی مجهول

همان مسئلهٔ انتقال حرارت یک‌بعدی از فصل پنجم را در نظر بگیرید، اما این بار ضریب نفوذ حرارتی $\alpha$ را نمی‌دانیم:

$$ \frac{\partial T}{\partial t} = \alpha \frac{\partial^2 T}{\partial x^2}, \quad (x,t) \in [0,1] \times [0,1] $$

شرایط مرزی (دیریکله):

$$ T(0,t) = 0, \quad T(1,t) = 0 $$

شرط اولیه:

$$ T(x,0) = \sin(\pi x) $$

اطلاعات در دسترس: داده‌های پراکنده‌ای از میدان دما در چند نقطهٔ خاص از دامنه (از سنسورهای حرارتی نصب‌شده روی قطعه).

هدف: با استفاده از داده‌های محدود دما در چند نقطه، هم میدان دما را در کل دامنه بازسازی کنید و هم مقدار $\alpha$ را تخمین بزنید.

مقدار واقعی $\alpha$ (برای مقایسه): $\alpha_{true} = 0.01$

ساختار ۵ بلوکی کد (تغییرات نسبت به فصل پنجم)

بلوک فصل پنجم (مستقیم) این فصل (معکوس) نوع تغییر
بلوک ۱ (شبکه) ورودی: ($x, t$) ورودی: ($x, t$) + پارامتر nn.Parameter برای $\alpha$ تغییر (اضافه شدن پارامتر یادگیرنده)
بلوک ۲ (نقاط) نقاط کالوکیشن + نقاط مرزی + شرط اولیه نقاط کالوکیشن + نقاط مرزی + شرط اولیه + نقاط داده تغییر (اضافه شدن نقاط داده)
بلوک ۳ (تلفات) $\mathcal{L} = \mathcal{L}_{physics} + \mathcal{L}_{BC} + \mathcal{L}_{IC}$ $\mathcal{L} = \mathcal{L}_{physics} + \mathcal{L}_{BC} + \mathcal{L}_{IC} + \mathcal{L}_{data}$ تغییر (اضافه شدن تلفات داده)
بلوک ۴ (آموزش) Adam → L-BFGS Adam → L-BFGS ثابت
بلوک ۵ (ارزیابی) مقایسه $T$ با جواب تحلیلی مقایسه $T$ با جواب تحلیلی + مقایسه $\alpha$ با مقدار واقعی تغییر (اضافه شدن گزارش تخمین α)

کد کامل PINN برای مسئلهٔ معکوس

# ════════════════════════════════════════════════════════════ # بلوک ۱: تعریف شبکه‌ی عصبی + پارامتر مجهول # ════════════════════════════════════════════════════════════ import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt class PINN_Inverse(nn.Module): def __init__(self, layers=[2, 50, 50, 50, 50, 1]): super(PINN_Inverse, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'tanh_{i}', nn.Tanh()) self.alpha = nn.Parameter(torch.tensor(0.05)) def forward(self, x, t): inputs = torch.cat([x, t], dim=1) return self.net(inputs) # ════════════════════════════════════════════════════════════ # بلوک ۲: تولید نقاط کالوکیشن، نقاط داده، و نقاط مرزی # ════════════════════════════════════════════════════════════ L, T = 1.0, 1.0 N_col = 5000 N_boundary = 200 N_ic = 200 N_data = 100 alpha_true = 0.01 x_col = torch.rand(N_col, 1, requires_grad=True) * L t_col = torch.rand(N_col, 1, requires_grad=True) * T t_boundary = torch.rand(N_boundary, 1) x_0 = torch.zeros(N_boundary, 1) x_1 = torch.ones(N_boundary, 1) x_ic = torch.rand(N_ic, 1) t_0 = torch.zeros(N_ic, 1) T_ic_true = torch.sin(np.pi * x_ic) x_data = torch.rand(N_data, 1) t_data = torch.rand(N_data, 1) T_data_true = torch.sin(np.pi * x_data) * torch.exp(-alpha_true * np.pi**2 * t_data) # ════════════════════════════════════════════════════════════ # بلوک ۳: تعریف تابع هزینه (با L_data اضافه شده) # ════════════════════════════════════════════════════════════ model = PINN_Inverse() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) def compute_loss(): T_pred = model(x_col, t_col) T_t = torch.autograd.grad(T_pred, t_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_x = torch.autograd.grad(T_pred, x_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_xx = torch.autograd.grad(T_x, x_col, grad_outputs=torch.ones_like(T_x), create_graph=True, retain_graph=True)[0] residual = T_t - model.alpha * T_xx loss_physics = torch.mean(residual**2) T_bc0 = model(x_0, t_boundary) loss_bc0 = torch.mean((T_bc0 - 0.0)**2) T_bc1 = model(x_1, t_boundary) loss_bc1 = torch.mean((T_bc1 - 0.0)**2) T_ic_pred = model(x_ic, t_0) loss_ic = torch.mean((T_ic_pred - T_ic_true)**2) T_data_pred = model(x_data, t_data) loss_data = torch.mean((T_data_pred - T_data_true)**2) loss = loss_physics + loss_bc0 + loss_bc1 + loss_ic + loss_data return loss, loss_physics, loss_bc0, loss_bc1, loss_ic, loss_data # ════════════════════════════════════════════════════════════ # بلوک ۴: حلقه‌ی آموزش (بدون تغییر) # ════════════════════════════════════════════════════════════ loss_history = [] alpha_history = [] n_epochs_adam = 10000 for epoch in range(n_epochs_adam): optimizer.zero_grad() loss, _, _, _, _, _ = compute_loss() loss.backward() optimizer.step() loss_history.append(loss.item()) alpha_history.append(model.alpha.item()) if (epoch + 1) % 2000 == 0: print(f'Epoch {epoch+1}/{n_epochs_adam}, Loss: {loss.item():.2e}, α: {model.alpha.item():.4f}') optimizer_lbfgs = torch.optim.LBFGS(model.parameters(), lr=1.0, max_iter=500) def closure(): optimizer_lbfgs.zero_grad() loss, _, _, _, _, _ = compute_loss() loss.backward() return loss optimizer_lbfgs.step(closure) # ════════════════════════════════════════════════════════════ # بلوک ۵: ارزیابی نتایج # ════════════════════════════════════════════════════════════ print(f'\nα_true: {alpha_true:.6f}') print(f'α_estimated: {model.alpha.item():.6f}') print(f'Relative error in α: {abs(model.alpha.item() - alpha_true) / alpha_true * 100:.2f}%') x_test = torch.linspace(0, L, 100).reshape(-1, 1) t_test = torch.linspace(0, T, 100).reshape(-1, 1) X, T_grid = torch.meshgrid(x_test.squeeze(), t_test.squeeze(), indexing='ij') x_flat = X.reshape(-1, 1) t_flat = T_grid.reshape(-1, 1) with torch.no_grad(): T_pred_flat = model(x_flat, t_flat).numpy() T_pred = T_pred_flat.reshape(X.shape) T_exact = np.sin(np.pi * X.numpy()) * np.exp(-alpha_true * np.pi**2 * T_grid.numpy()) error_l2 = np.linalg.norm(T_pred - T_exact) / np.linalg.norm(T_exact) print(f'L2 Error in temperature field: {error_l2:.2e}') fig, axes = plt.subplots(2, 2, figsize=(14, 10)) im1 = axes[0, 0].contourf(T_grid, X, T_exact, levels=50, cmap='hot') axes[0, 0].set_xlabel('Time t', fontsize=12) axes[0, 0].set_ylabel('Position x', fontsize=12) axes[0, 0].set_title('Analytical Solution', fontsize=14) plt.colorbar(im1, ax=axes[0, 0]) im2 = axes[0, 1].contourf(T_grid, X, T_pred, levels=50, cmap='hot') axes[0, 1].set_xlabel('Time t', fontsize=12) axes[0, 1].set_ylabel('Position x', fontsize=12) axes[0, 1].set_title('PINN Prediction', fontsize=14) plt.colorbar(im2, ax=axes[0, 1]) im3 = axes[1, 0].contourf(T_grid, X, np.abs(T_pred - T_exact), levels=50, cmap='coolwarm') axes[1, 0].set_xlabel('Time t', fontsize=12) axes[1, 0].set_ylabel('Position x', fontsize=12) axes[1, 0].set_title('Absolute Error', fontsize=14) plt.colorbar(im3, ax=axes[1, 0]) ax4 = axes[1, 1] ax4_alpha = ax4.twinx() ax4.semilogy(loss_history, color='blue', label='Loss') ax4_alpha.plot(alpha_history, color='red', linestyle='--', label='α estimated') ax4_alpha.axhline(y=alpha_true, color='green', linestyle=':', linewidth=2, label='α_true') ax4.set_xlabel('Epoch', fontsize=12) ax4.set_ylabel('Loss', fontsize=12, color='blue') ax4_alpha.set_ylabel('α', fontsize=12, color='red') ax4.set_title('Loss and α Estimation History', fontsize=14) ax4.grid(True, alpha=0.3) ax4.legend(loc='upper left') ax4_alpha.legend(loc='upper right') plt.tight_layout() plt.show()

تشریح تغییرات کلیدی در کد

تعریف پارامتر مجهول به‌عنوان nn.Parameter (بلوک ۱)

self.alpha = nn.Parameter(torch.tensor(0.05))

در PyTorch، nn.Parameter یک Tensor است که به‌طور خودکار به لیست پارامترهای مدل اضافه می‌شود. این ویژگی، هستهٔ قدرت PINN در مسائل معکوس است.

اضافه شدن نقاط داده (بلوک ۲)

x_data = torch.rand(N_data, 1) t_data = torch.rand(N_data, 1) T_data_true = torch.sin(np.pi * x_data) * torch.exp(-alpha_true * np.pi**2 * t_data)

اضافه شدن تلفات داده ($\mathcal{L}_{data}$) (بلوک ۳)

T_data_pred = model(x_data, t_data) loss_data = torch.mean((T_data_pred - T_data_true)**2)

بدون این جمله، مسئلهٔ معکوس به یک مسئلهٔ مستقیم با $\alpha$ مجهول تبدیل می‌شود که بی‌نهایت جواب مختلف دارد.

شکل ۶-۲: تاریخچهٔ همگرایی α (خط قرمز) در کنار تلفات (خط آبی) — تخمین ضریب مجهول با استفاده از داده‌های پراکنده

گسترش به سیستم‌های غیرخطی

مثال انتقال حرارت یک مسئلهٔ خطی بود. اما در دنیای واقعی، به‌ویژه در سیستم‌های ارتعاشی بوژی‌های قطار، با پدیده‌های غیرخطی مواجه هستیم. در این بخش، نشان می‌دهیم که چگونه PINN می‌تواند به‌سادگی با اضافه کردن یک nn.Parameter جدید، ضرایب غیرخطی را نیز شناسایی کند.

سیستم جرم-فنر-دمپر غیرخطی (مدل بوژی ساده‌شده):

$$ m\ddot{x} + c_1\dot{x} + c_2|\dot{x}|\dot{x} + kx = 0, \quad x(0) = 1, \quad \dot{x}(0) = 0 $$

تغییرات کد نسبت به مثال خطی:

# در بلوک ۱ (تعریف شبکه) self.c1 = nn.Parameter(torch.tensor(0.5)) self.c2 = nn.Parameter(torch.tensor(0.1)) self.k = nn.Parameter(torch.tensor(5.0)) # در بلوک ۳ (تابع هزینه) x_pred = model(t_col) dx = torch.autograd.grad(x_pred, t_col, grad_outputs=torch.ones_like(x_pred), create_graph=True, retain_graph=True)[0] d2x = torch.autograd.grad(dx, t_col, grad_outputs=torch.ones_like(dx), create_graph=True, retain_graph=True)[0] residual = m * d2x + model.c1 * dx + model.c2 * torch.abs(dx) * dx + model.k * x_pred loss_physics = torch.mean(residual**2)

جمع‌بندی و مسیر پیش‌رو

در این فصل، با یکی از قدرتمندترین کاربردهای PINN، یعنی مسائل معکوس، آشنا شدیم. نکات کلیدی عبارت‌اند از:

  1. تعریف پارامتر مجهول: با استفاده از nn.Parameter، پارامترهای فیزیکی مجهول را به‌عنوان بخشی از مدل تعریف می‌کنیم.
  2. اضافه شدن تلفات داده ($\mathcal{L}_{data}$): برای تطابق شبکه با داده‌های اندازه‌گیری، یک جملهٔ تلفات جدید به تابع هزینه اضافه می‌شود.
  3. توانایی تخمین هم‌زمان: شبکه به‌طور هم‌زمان هم میدان مجهول و هم پارامترهای فیزیکی مجهول را یاد می‌گیرد.
  4. گسترش به سیستم‌های غیرخطی: با اضافه کردن پارامترهای جدید و اصلاح تلفات فیزیک، PINN به‌راحتی می‌تواند ضرایب غیرخطی را نیز شناسایی کند.

پل به فصل هفتم: در فصل بعد، خواهیم دید که اگر پارامتر را به‌عنوان ورودی شبکه تعریف کنیم (نه پارامتر یادگیرنده)، چه قابلیت‌های شگفت‌انگیزی در طراحی بهینه و تحلیل سریع به‌دست می‌آید.

تمرین‌های عملی

  1. تمرین ۱ (تأثیر تعداد داده‌ها بر دقت تخمین α): کد را با تعداد نقاط دادهٔ $N_{data} = 5, 20, 50, 200$ اجرا کنید.
  2. تمرین ۲ (تأثیر نویز بر تخمین α): به داده‌های شبیه‌سازی‌شده نویز گاوسی با انحراف معیار ۰.۰۱ اضافه کنید.
  3. تمرین ۳ (تأثیر نوع داده‌ها: گذرا در برابر ماندگار): داده‌ها را فقط از زمان‌های بزرگ (مثلاً $t_{data} = 0.8 + 0.2 \times \text{rand}(N_{data}, 1)$) برداشت کنید.
