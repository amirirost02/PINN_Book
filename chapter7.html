فصل ۷: مسائل پارامتریک — حل خانواده‌ای از مسائل با یک بار آموزش

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. تفاوت بنیادین بین «مسئلهٔ معکوس» و «مسئلهٔ پارامتریک» را در زمینهٔ PINN تشریح کنید.
  2. یک پارامتر فیزیکی (مانند ضریب نفوذ حرارتی یا سختی فنر) را به‌عنوان ورودی شبکه (نه پارامتر یادگیرنده) اضافه کنید.
  3. یک PINN پارامتریک را برای حل خانواده‌ای از مسائل با مقادیر مختلف پارامتر، با یک بار آموزش، پیاده‌سازی کنید.
  4. تأثیر افزودن یک بُعد پارامتری به ورودی شبکه را بر معماری و فرآیند آموزش تحلیل کنید.
  5. از شبکهٔ آموزش‌دیده برای پیش‌بینی سریع پاسخ برای مقادیر جدید پارامتر (بدون آموزش مجدد) استفاده کنید.

از «شناسایی یک پارامتر» تا «حل برای همهٔ پارامترها»

در فصل ششم، با مفهوم مسائل معکوس آشنا شدیم. در آنجا، یک پارامتر مجهول (ضریب نفوذ حرارتی $\alpha$) را با استفاده از داده‌های پراکنده شناسایی کردیم. اما سناریوی دیگری نیز در مهندسی — به‌ویژه در طراحی و تحلیل سیستم‌های ریلی — بسیار رایج است: می‌خواهیم پاسخ یک سیستم را برای مقادیر مختلف یک پارامتر (مثلاً سختی فنر بوژی، ضریب میرایی، یا سرعت قطار) پیش‌بینی کنیم.

برای مثال، در طراحی یک بوژی جدید، مهندس ممکن است بخواهد تأثیر تغییر سختی فنرهای ثانویه را بر روی پاسخ ارتعاشی واگن بررسی کند. روش کلاسیک این است که برای هر مقدار سختی، یک بار کامل تحلیل المان محدود اجرا شود. این کار برای ۱۰۰ مقدار مختلف سختی، به ۱۰۰ بار اجرای جداگانه نیاز دارد.

راه‌حل: مسائل پارامتریک (Parametric Problems)
در این رویکرد، پارامتر متغیر را به‌عنوان یک ورودی اضافی به شبکه معرفی می‌کنیم. شبکه با دیدن مقادیر مختلف این پارامتر در طول آموزش، یاد می‌گیرد که چگونه پاسخ سیستم به تغییرات پارامتر وابسته است. پس از پایان آموزش، شبکه می‌تواند برای هر مقدار جدیدی از پارامتر (حتی مقادیری که در آموزش ندیده است)، پاسخ را در کسری از ثانیه پیش‌بینی کند.

شکل ۷-۱: مقایسهٔ شماتیک جایگاه پارامتر در مسئلهٔ معکوس (چپ) و مسئلهٔ پارامتریک (راست)
ویژگی مسئلهٔ معکوس (فصل ۶) مسئلهٔ پارامتریک (این فصل)
جایگاه پارامتر پارامتر یادگیرنده (nn.Parameter) ورودی شبکه (Input)
هدف شناسایی یک مقدار خاص برای پارامتر یادگیری وابستگی پاسخ به پارامتر
تعداد پارامترها یک مقدار ثابت (تخمین‌زده می‌شود) هر مقدار دلخواه در یک بازه
کاربرد اصلی شناسایی خواص ماده از روی داده طراحی بهینه، تحلیل سریع، آنالیز حساسیت
آموزش مجدد برای پارامتر جدید خیر خیر (شبکه قبلاً همهٔ حالات را دیده است)

معرفی مسئله: انتقال حرارت با ضریب نفوذ حرارتی متغیر

همان مسئلهٔ انتقال حرارت یک‌بعدی از فصل پنجم را در نظر بگیرید، اما این بار ضریب نفوذ حرارتی $\alpha$ را به‌عنوان یک ورودی به شبکه اضافه می‌کنیم:

$$ \frac{\partial T}{\partial t} = \alpha \frac{\partial^2 T}{\partial x^2}, \quad (x,t) \in [0,1] \times [0,1] $$

شرایط مرزی (دیریکله):

$$ T(0,t) = 0, \quad T(1,t) = 0 $$

شرط اولیه:

$$ T(x,0) = \sin(\pi x) $$

پارامتر متغیر (ورودی شبکه):

$$ \alpha \in [0.005, 0.02] $$

هدف: یک PINN پارامتریک آموزش دهید که بتواند برای هر مقدار $\alpha$ در بازهٔ مشخص شده، میدان دما را پیش‌بینی کند.

ساختار ۵ بلوکی کد (تغییرات نسبت به فصل پنجم)

بلوک فصل پنجم (مستقیم با α ثابت) این فصل (پارامتریک) نوع تغییر
بلوک ۱ (شبکه) ورودی: ($x, t$) ورودی: ($x, t, \alpha$) تغییر (افزایش بُعد ورودی به ۳)
بلوک ۲ (نقاط) نقاط برای ($x, t$) با α ثابت نقاط برای ($x, t, \alpha$) در فضای سه‌بعدی تغییر (تولید نقاط با α متغیر)
بلوک ۳ (تلفات) $\mathcal{L} = \mathcal{L}_{physics} + \mathcal{L}_{BC} + \mathcal{L}_{IC}$ $\mathcal{L} = \mathcal{L}_{physics} + \mathcal{L}_{BC} + \mathcal{L}_{IC}$ ثابت (ساختار تلفات بدون تغییر)
بلوک ۴ (آموزش) Adam → L-BFGS Adam → L-BFGS ثابت
بلوک ۵ (ارزیابی) مقایسه برای یک α مشخص مقایسه برای چندین α مختلف تغییر (ارزیابی چندگانه)

کد کامل PINN برای مسئلهٔ پارامتریک

# ════════════════════════════════════════════════════════════ # بلوک ۱: تعریف شبکه‌ی عصبی با ورودی ۳ بعدی (x, t, α) # ════════════════════════════════════════════════════════════ import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt class PINN_Parametric(nn.Module): def __init__(self, layers=[3, 50, 50, 50, 50, 1]): super(PINN_Parametric, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'tanh_{i}', nn.Tanh()) def forward(self, x, t, alpha): inputs = torch.cat([x, t, alpha], dim=1) return self.net(inputs) # ════════════════════════════════════════════════════════════ # بلوک ۲: تولید نقاط کالوکیشن و نقاط داده در فضای سه‌بعدی # ════════════════════════════════════════════════════════════ L, T = 1.0, 1.0 alpha_min, alpha_max = 0.005, 0.02 N_col = 10000 N_boundary = 500 N_ic = 500 x_col = torch.rand(N_col, 1, requires_grad=True) * L t_col = torch.rand(N_col, 1, requires_grad=True) * T alpha_col = torch.rand(N_col, 1) * (alpha_max - alpha_min) + alpha_min t_boundary = torch.rand(N_boundary, 1) alpha_boundary = torch.rand(N_boundary, 1) * (alpha_max - alpha_min) + alpha_min x_0 = torch.zeros(N_boundary, 1) x_1 = torch.ones(N_boundary, 1) x_ic = torch.rand(N_ic, 1) alpha_ic = torch.rand(N_ic, 1) * (alpha_max - alpha_min) + alpha_min t_0 = torch.zeros(N_ic, 1) T_ic_true = torch.sin(np.pi * x_ic) # ════════════════════════════════════════════════════════════ # بلوک ۳: تعریف تابع هزینه (ساختار مشابه فصل پنجم) # ════════════════════════════════════════════════════════════ model = PINN_Parametric() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) def compute_loss(): T_pred = model(x_col, t_col, alpha_col) T_t = torch.autograd.grad(T_pred, t_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_x = torch.autograd.grad(T_pred, x_col, grad_outputs=torch.ones_like(T_pred), create_graph=True, retain_graph=True)[0] T_xx = torch.autograd.grad(T_x, x_col, grad_outputs=torch.ones_like(T_x), create_graph=True, retain_graph=True)[0] residual = T_t - alpha_col * T_xx loss_physics = torch.mean(residual**2) T_bc0 = model(x_0, t_boundary, alpha_boundary) loss_bc0 = torch.mean((T_bc0 - 0.0)**2) T_bc1 = model(x_1, t_boundary, alpha_boundary) loss_bc1 = torch.mean((T_bc1 - 0.0)**2) T_ic_pred = model(x_ic, t_0, alpha_ic) loss_ic = torch.mean((T_ic_pred - T_ic_true)**2) loss = loss_physics + loss_bc0 + loss_bc1 + loss_ic return loss, loss_physics, loss_bc0, loss_bc1, loss_ic # ════════════════════════════════════════════════════════════ # بلوک ۴: حلقه‌ی آموزش # ════════════════════════════════════════════════════════════ loss_history = [] n_epochs_adam = 15000 for epoch in range(n_epochs_adam): optimizer.zero_grad() loss, _, _, _, _ = compute_loss() loss.backward() optimizer.step() loss_history.append(loss.item()) if (epoch + 1) % 3000 == 0: print(f'Epoch {epoch+1}/{n_epochs_adam}, Loss: {loss.item():.2e}') optimizer_lbfgs = torch.optim.LBFGS(model.parameters(), lr=1.0, max_iter=500) def closure(): optimizer_lbfgs.zero_grad() loss, _, _, _, _ = compute_loss() loss.backward() return loss optimizer_lbfgs.step(closure) # ════════════════════════════════════════════════════════════ # بلوک ۵: ارزیابی برای چندین مقدار α # ════════════════════════════════════════════════════════════ alpha_test_values = [0.005, 0.007, 0.01, 0.015, 0.02] x_test = torch.linspace(0, L, 80).reshape(-1, 1) t_test = torch.linspace(0, T, 80).reshape(-1, 1) X, T_grid = torch.meshgrid(x_test.squeeze(), t_test.squeeze(), indexing='ij') x_flat = X.reshape(-1, 1) t_flat = T_grid.reshape(-1, 1) fig, axes = plt.subplots(2, 3, figsize=(15, 8)) axes = axes.flatten() for idx, alpha_val in enumerate(alpha_test_values): alpha_flat = torch.full_like(x_flat, alpha_val) with torch.no_grad(): T_pred_flat = model(x_flat, t_flat, alpha_flat).numpy() T_pred = T_pred_flat.reshape(X.shape) T_exact = np.sin(np.pi * X.numpy()) * np.exp(-alpha_val * np.pi**2 * T_grid.numpy()) error_l2 = np.linalg.norm(T_pred - T_exact) / np.linalg.norm(T_exact) im = axes[idx].contourf(T_grid, X, T_pred, levels=30, cmap='hot') axes[idx].set_xlabel('Time t', fontsize=10) axes[idx].set_ylabel('Position x', fontsize=10) axes[idx].set_title(f'α = {alpha_val:.3f}, L2 Error = {error_l2:.2e}', fontsize=10) plt.colorbar(im, ax=axes[idx]) fig.delaxes(axes[5]) plt.tight_layout() plt.show() print("\n=== Performance for different α values ===") for alpha_val in alpha_test_values: alpha_flat = torch.full_like(x_flat, alpha_val) with torch.no_grad(): T_pred_flat = model(x_flat, t_flat, alpha_flat).numpy() T_pred = T_pred_flat.reshape(X.shape) T_exact = np.sin(np.pi * X.numpy()) * np.exp(-alpha_val * np.pi**2 * T_grid.numpy()) error_l2 = np.linalg.norm(T_pred - T_exact) / np.linalg.norm(T_exact) print(f'α = {alpha_val:.4f}, L2 Error = {error_l2:.2e}')

تشریح تغییرات کلیدی در کد

افزایش بُعد ورودی شبکه (بلوک ۱)

در این مسئله، شبکه باید سه ورودی (مکان $x$، زمان $t$، و ضریب نفوذ حرارتی $\alpha$) را دریافت کند. لایهٔ اول شبکه با 3 نورون شروع می‌شود: layers=[3, 50, 50, 50, 50, 1].

تولید نقاط در فضای سه‌بعدی (بلوک ۲)

$\alpha$ در بازهٔ $[0.005, 0.02]$ به‌صورت یکنواخت نمونه‌برداری می‌شود. $\alpha$ به‌عنوان ورودی شبکه و همچنین در محاسبهٔ باقیماندهٔ معادله استفاده می‌شود، اما مشتق $T$ نسبت به $\alpha$ محاسبه نمی‌شود.

تلفات فیزیک با $\alpha$ متغیر (بلوک ۳)

residual = T_t - alpha_col * T_xx
شکل ۷-۲: پیش‌بینی PINN پارامتریک برای ۵ مقدار مختلف α — خطا برای هر مقدار نشان داده شده است

تحلیل نتایج و مزایای رویکرد پارامتریک

  1. خطای L2 برای مقادیر مختلف $\alpha$: خطا برای مقادیر دیده‌شده در آموزش (مثلاً $\alpha=0.01$) بسیار کوچک است (حدود $10^{-5}$). برای مقادیر دیده‌نشده (مثلاً $\alpha=0.007$) نیز خطا همچنان کوچک است (حدود $10^{-4}$).
  2. محدودیت‌های برون‌یابی (Extrapolation): اگر از شبکه برای مقادیر خارج از بازهٔ آموزشی (مثلاً $\alpha=0.001$ یا $\alpha=0.03$) استفاده کنیم، خطا به‌سرعت افزایش می‌یابد.
  3. سرعت پیش‌بینی: پس از آموزش، پیش‌بینی برای هر مقدار جدید $\alpha$ در کمتر از یک ثانیه انجام می‌شود.
  4. کاربرد در طراحی بهینه: با داشتن یک شبکهٔ پارامتریک، می‌توان به‌سرعت تأثیر تغییر پارامترها را بر پاسخ سیستم بررسی کرد.

جمع‌بندی و مسیر پیش‌رو

در این فصل، با مفهوم مسائل پارامتریک آشنا شدیم. نکات کلیدی عبارت‌اند از:

  1. تفاوت با مسئلهٔ معکوس: در مسائل پارامتریک، پارامتر به‌عنوان ورودی شبکه (نه پارامتر یادگیرنده) تعریف می‌شود.
  2. افزایش بُعد ورودی: با اضافه کردن یک بُعد پارامتری به ورودی شبکه، شبکه می‌تواند خانواده‌ای از مسائل را با یک بار آموزش حل کند.
  3. قابلیت تعمیم‌پذیری: شبکهٔ آموزش‌دیده می‌تواند برای مقادیر جدید پارامتر (در بازهٔ آموزشی) پاسخ‌های دقیقی تولید کند.
  4. محدودیت‌ها: افزایش تعداد پارامترها هزینهٔ آموزش را افزایش می‌دهد و شبکه در برون‌یابی عملکرد ضعیفی دارد.

پل به فصل هشتم: در این فصل دیدیم که چگونه با تعریف پارامترها به‌عنوان ورودی شبکه، می‌توان خانواده‌ای از مسائل را حل کرد. در فصل هشتم، از همین ایده برای حل مسائل مقدار ویژه (تحلیل مودال) استفاده خواهیم کرد.

تمرین‌های عملی

  1. تمرین ۱ (تأثیر بازهٔ α بر دقت و تعمیم‌پذیری): بازهٔ α را از $[0.005, 0.02]$ به $[0.001, 0.05]$ افزایش دهید.
  2. تمرین ۲ (برون‌یابی): با استفاده از شبکهٔ آموزش‌دیده در بازهٔ $[0.005, 0.02]$، خطا را برای $\alpha=0.04$ (خارج از بازه) محاسبه کنید.
  3. تمرین ۳ (پارامتر جدید — شرط مرزی متغیر): شرط مرزی را از $T(0,t)=0$ به $T(0,t)=T_{left}$ تغییر دهید که $T_{left}$ یک پارامتر متغیر در بازهٔ $[0, 2]$ است.
