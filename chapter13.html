فصل ۱۳: چالش سوم — مسائل معکوس بدحالت، نویز، و پروژه نهایی

اهداف یادگیری

پس از مطالعهٔ این فصل، شما قادر خواهید بود:

  1. مفهوم «بدحالت‌بودن» (Ill-posedness) را در مسائل معکوس تعریف کرده و نقش نویز را در آن تحلیل کنید.
  2. تأثیر نویز گاوسی را بر دقت تخمین پارامترهای مجهول با استفاده از PINN بررسی کنید.
  3. تفاوت بین «منظم‌سازی روی پارامترهای فیزیکی» و «وزن‌کاهی روی وزن‌های شبکه» را توضیح دهید.
  4. روش منظم‌سازی L2 را برای کاهش اثر نویز در تخمین پارامتر پیاده‌سازی کنید.
  5. مفهوم «عدم یکتایی» (Non-Uniqueness) را توضیح داده و نقش داده‌های گذرا در رفع آن را تشریح کنید.
  6. یک پروژهٔ کامل PINN معکوس را برای شناسایی هم‌زمان چند پارامتر مجهول با داده‌های نویزی، از صفر تا صد پیاده‌سازی کنید.

وقتی داده‌ها کامل نیستند — بدحالتی و نویز

در دنیای واقعی، داده‌ها هرگز کامل نیستند. عوامل متعددی مانند نویز سنسورها، داده‌های محدود، و مدل‌های تقریب‌زده، مسئلهٔ معکوس را به یک مسئلهٔ بدحالت (Ill-posed Problem) تبدیل می‌کنند.

تعریف بدحالتی (Hadamard):
یک مسئلهٔ معکوس بدحالت است اگر:

  1. جوابی وجود نداشته باشد،
  2. جواب یکتا نباشد، یا
  3. جواب به‌شدت به تغییرات کوچک در داده‌ها حساس باشد.
شکل ۱۳-۱: مقایسهٔ تخمین پارامتر با داده‌های بدون نویز (چپ) و داده‌های نویزی (راست) — نویز باعث انحراف تخمین می‌شود

تأثیر نویز بر تخمین پارامتر

شبیه‌سازی نویز در داده‌ها

$$ T_{data}^{noisy}(x_i, t_i) = T_{data}^{exact}(x_i, t_i) + \eta \cdot \sigma \cdot \mathcal{N}(0,1) $$

علت: اورفیتینگ به نویز

وقتی داده‌ها دارای نویز هستند، شبکه برای کاهش $\mathcal{L}_{data}$، سعی می‌کند نویز را نیز بازتولید کند و تخمین پارامتر منحرف می‌شود.

منظم‌سازی برای مقابله با نویز

تفکیک دو نوع منظم‌سازی

نوع منظم‌سازی عبارت ریاضی هدف کاربرد
منظم‌سازی روی پارامترهای فیزیکی $\lambda_{reg} \cdot \sum_{p \in \mathcal{P}} p^2$ جلوگیری از بزرگ‌شدن پارامترها مسائل با پارامترهای کم
وزن‌کاهی روی وزن‌های شبکه $\lambda_{wd} \cdot \sum_{w} w^2$ هموارسازی پاسخ شبکه مسائل با میدان‌های پیچیده

منظم‌سازی L2 روی پارامترهای فیزیکی (Tikhonov)

$$ \mathcal{L}_{total} = \mathcal{L}_{physics} + \mathcal{L}_{data} + \lambda_{reg} \cdot (c^2 + k^2) $$
شکل ۱۳-۲: تأثیر منظم‌سازی L2 بر تخمین پارامترهای $c$ و $k$ — خطای تخمین با افزایش $\lambda_{reg}$ کاهش می‌یابد

پروژهٔ نهایی: شناسایی هم‌زمان ضریب میرایی و سختی

تعریف مسئله

سیستم جرم-فنر-دمپر:

$$ m\ddot{x} + c\dot{x} + kx = 0, \quad x(0) = 1, \quad \dot{x}(0) = 0 $$

پارامترهای مجهول: $c$ و $k$ ($m = 1$)
داده‌ها: شتاب $\ddot{x}(t)$ در ۲۰۰ نقطه با نویز ۳\%
هدف: تخمین هم‌زمان $c$ و $k$

کد کامل پروژه

# ════════════════════════════════════════════════════════════ # پروژه نهایی: شناسایی هم‌زمان c و k با داده‌های نویزی # ════════════════════════════════════════════════════════════ import torch import torch.nn as nn import numpy as np import matplotlib.pyplot as plt torch.set_default_dtype(torch.float64) class PINN_MSD(nn.Module): def __init__(self, layers=[1, 50, 50, 50, 1]): super(PINN_MSD, self).__init__() self.net = nn.Sequential() for i in range(len(layers) - 1): self.net.add_module(f'linear_{i}', nn.Linear(layers[i], layers[i+1])) if i < len(layers) - 2: self.net.add_module(f'tanh_{i}', nn.Tanh()) self.c = nn.Parameter(torch.tensor(1.0)) self.k = nn.Parameter(torch.tensor(5.0)) def forward(self, t): return self.net(t) m_true = torch.tensor(1.0) c_true = torch.tensor(0.5) k_true = torch.tensor(10.0) zeta = (c_true / (2 * torch.sqrt(k_true * m_true))).item() omega_n = torch.sqrt(k_true / m_true).item() omega_d = omega_n * np.sqrt(1 - zeta**2) def analytical_solution(t): return np.exp(-zeta * omega_n * t) * (np.cos(omega_d * t) + (zeta * omega_n / omega_d) * np.sin(omega_d * t)) N_data = 200 t_data = torch.rand(N_data, 1, requires_grad=True) * 10 t_np = t_data.detach().numpy() x_data = torch.tensor(analytical_solution(t_np), dtype=torch.float64) v_data = -zeta * omega_n * x_data + np.exp(-zeta * omega_n * t_np) * ( -omega_d * np.sin(omega_d * t_np) + (zeta * omega_n / omega_d) * omega_d * np.cos(omega_d * t_np)) a_data = -zeta * omega_n * v_data - omega_n**2 * x_data noise_level = 0.03 noise = noise_level * torch.randn_like(a_data) a_data_noisy = a_data + noise N_col = 500 t_col = torch.rand(N_col, 1, requires_grad=True) * 10 model = PINN_MSD() optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) def compute_loss(t_col, t_data, a_data_noisy): x = model(t_col) dx = torch.autograd.grad(x, t_col, grad_outputs=torch.ones_like(x), create_graph=True, retain_graph=True)[0] d2x = torch.autograd.grad(dx, t_col, grad_outputs=torch.ones_like(dx), create_graph=True, retain_graph=True)[0] ode_residual = m_true * d2x + model.c * dx + model.k * x loss_ode = torch.mean(ode_residual**2) t_ic = torch.tensor([[0.0]], requires_grad=True) x_ic = model(t_ic) dx_ic = torch.autograd.grad(x_ic, t_ic, grad_outputs=torch.ones_like(x_ic), create_graph=True, retain_graph=True)[0] loss_ic = (x_ic - 1.0)**2 + (dx_ic - 0.0)**2 x_data_pred = model(t_data) dx_data = torch.autograd.grad(x_data_pred, t_data, grad_outputs=torch.ones_like(x_data_pred), create_graph=True, retain_graph=True)[0] d2x_data = torch.autograd.grad(dx_data, t_data, grad_outputs=torch.ones_like(dx_data), create_graph=True, retain_graph=True)[0] a_pred = m_true * d2x_data + model.c * dx_data + model.k * x_data_pred loss_data = torch.mean((a_pred - a_data_noisy)**2) lambda_reg = 1e-5 loss_reg = lambda_reg * (model.c**2 + model.k**2) loss = loss_ode + loss_ic + loss_data + loss_reg return loss, loss_ode, loss_ic, loss_data, loss_reg loss_history = [] c_history = [] k_history = [] n_epochs = 8000 for epoch in range(n_epochs): optimizer.zero_grad() loss, _, _, _, _ = compute_loss(t_col, t_data, a_data_noisy) loss.backward() optimizer.step() loss_history.append(loss.item()) c_history.append(model.c.item()) k_history.append(model.k.item()) if (epoch+1) % 2000 == 0: print(f'Epoch {epoch+1}: Loss={loss.item():.2e}, ' f'c={model.c.item():.4f}, k={model.k.item():.4f}') optimizer_lbfgs = torch.optim.LBFGS(model.parameters(), lr=1.0, max_iter=500) def closure(): optimizer_lbfgs.zero_grad() loss, _, _, _, _ = compute_loss(t_col, t_data, a_data_noisy) loss.backward() return loss optimizer_lbfgs.step(closure) print(f'\nTrue c: {c_true.item():.4f}, Estimated c: {model.c.item():.4f}') print(f'True k: {k_true.item():.4f}, Estimated k: {model.k.item():.4f}') print(f'Error in c: {abs(model.c.item() - c_true.item()) / c_true.item() * 100:.2f}%') print(f'Error in k: {abs(model.k.item() - k_true.item()) / k_true.item() * 100:.2f}%') t_test = torch.linspace(0, 10, 500).reshape(-1, 1) with torch.no_grad(): x_pred = model(t_test).numpy() x_exact = analytical_solution(t_test.numpy()) plt.figure(figsize=(14, 5)) plt.subplot(1, 3, 1) plt.plot(t_test, x_exact, 'b-', label='Exact') plt.plot(t_test, x_pred, 'r--', label='PINN') plt.xlabel('Time t') plt.ylabel('Displacement x(t)') plt.legend() plt.grid(True) plt.subplot(1, 3, 2) plt.semilogy(loss_history) plt.xlabel('Epoch') plt.ylabel('Loss') plt.grid(True) plt.title('Loss History') plt.subplot(1, 3, 3) plt.plot(c_history, label='Estimated c') plt.axhline(y=c_true.item(), color='r', linestyle='--', label='True c') plt.plot(k_history, label='Estimated k') plt.axhline(y=k_true.item(), color='b', linestyle='--', label='True k') plt.xlabel('Epoch') plt.ylabel('Parameter value') plt.legend() plt.grid(True) plt.tight_layout() plt.show()

جمع‌بندی و چشم‌انداز پایان دوره

در این فصل، با چالش‌های نهایی مسائل معکوس آشنا شدیم و پروژهٔ نهایی را با موفقیت به پایان رساندیم.

دستاوردهای شما:

  1. درک عمیق از PINN و تفاوت آن با شبکه‌های عصبی معمولی.
  2. توانایی پیاده‌سازی PINN برای مسائل مستقیم، معکوس، پارامتریک، و مودال.
  3. تسلط بر ترفندهای عیب‌زدایی (float64، نرمال‌سازی، وزن‌دهی پویا، و آموزش گام‌به‌گام).
  4. آشنایی با روش‌های پیشرفتهٔ اعمال شرایط مرزی (SDF)، نمونه‌برداری تطبیقی (RAR)، و معماری‌های مقاوم (SIREN).
  5. توانایی مواجهه با داده‌های نویزی و استفاده از منظم‌سازی.

پل به فصل چهاردهم: در فصل بعد، با حوزه‌های پیشرفته‌تر مانند یادگیری عملگرها (DeepONet و FNO) و PINN بیزی (B-PINN) آشنا خواهید شد.

تمرین‌های عملی

  1. تمرین ۱: پروژه را با سطوح نویز ۱\%، ۵\%، و ۱۰\% اجرا کنید.
  2. تمرین ۲: پروژه را با $\lambda_{reg} = 0$، $10^{-6}$، $10^{-5}$، و $10^{-4}$ اجرا کنید.
  3. تمرین ۳: داده‌ها را فقط از بازهٔ $[5,10]$ (حالت ماندگار) انتخاب کنید.
